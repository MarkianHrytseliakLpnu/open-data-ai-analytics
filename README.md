# Open Data AI Analytics: Міжнародна Технічна Допомога

Проєкт автоматизованого конвеєра (pipeline) для завантаження, обробки, аналізу якості, дослідження та візуалізації відкритих даних щодо міжнародної технічної допомоги в Україні. Проєкт використовує мікросервісну архітектуру на базі Docker.

---

## 📊 Про датасет

Дані взято з державного порталу наборів даних [data.gov.ua](https://data.gov.ua/).

**Набір даних:** [Інформація про отримане майно (обладнання, програмне забезпечення) у рамках міжнародної технічної допомоги](https://data.gov.ua/dataset/scsu-information-about-received-property)

---

## 💡 Гіпотези щодо даних

1. Які проєкти (`projectTitle`) отримали найбільшу сумарну вартість активів, і чи існує значна диспропорція ("перекіс") у фінансуванні певних сфер (наприклад, кібербезпека vs кордон)?
2. Чи є на ринку донорів домінуючі організації (`implementerName`), які забезпечують левову частку поставок, чи допомога диверсифікована між багатьма виконавцями?
3. Як змінювалася інтенсивність надання допомоги (кількість передач або їх вартість) у часі? Чи є пікові періоди (місяці), коли надходило найбільше обладнання?

## 🏗 Структура проєкту

Проєкт організовано за принципом мікросервісів, де кожен етап пайплайну працює у власному ізольованому Docker-контейнері:

```text
open-data-ai-analytics/
├── data/                 # Спільний Volume для збереження датасетів (raw)
├── docker/               # Dockerfiles та залежності для кожного сервісу
│   ├── loader/           
│   ├── quality/
│   ├── research/
│   ├── visualization/
│   └── web/
├── reports/              # Спільний Volume для збереження звітів (json, png)
├── src/                  # Вихідний код пайплайну
│   ├── config.py         # Глобальні налаштування та шляхи
│   ├── loader         # Завантаження CSV з data.gov.ua, Очищення даних та збереження в PostgreSQL
│   ├── quality        # Аналіз якості даних (missing values, duplicates)
│   ├── research       # Статистичний аналіз та пошук інсайтів
│   ├── visualization  # Побудова графіків (matplotlib/seaborn)
│   └── web            # Streamlit веб-інтерфейс
├── tests/                # Модульні тести (Pytest)
├── docker-compose.yml    # Головний конфіг розгортання
└── pytest.ini            # Налаштування для Pytest 
```

## 🚀 Команди запуску

Для розгортання всього проєкту потрібен лише встановлений **Docker** та **Docker Compose**.

### 1. Повний запуск пайплайну
Щоб зібрати образи, запустити базу даних, виконати всі етапи аналітики та підняти веб-інтерфейс, виконайте команду в корені проєкту:
```bash
docker-compose up --build
```

### 2. Запуск у фоновому режимі (Detached mode)
Щоб звільнити термінал після запуску:

```bash
docker-compose up -d --build
```

### 3. Зупинка проєкту
Щоб зупинити всі контейнери (дані бази даних та завантажені звіти збережуться завдяки Volumes):

```bash
docker-compose down
```

## 📦 Опис сервісів (Docker Containers)

Архітектура `docker-compose.yml` включає наступні сервіси:

### 1. Інфраструктурні (Працюють постійно)
* **`db` (PostgreSQL):** СУБД для надійного зберігання очищених даних.
* **`pgadmin`:** Зручний веб-інтерфейс для адміністрування бази даних та виконання SQL-запитів.
* **`web` (Streamlit App):** Дашборд для інтерактивного перегляду результатів (даних, звітів, візуалізацій).

### 2. Аналітичні Batch-сервіси (Відпрацювали та вимкнулися)
* **`loader`:** Завантажує `.csv` файл з API, очищує типи даних та зберігає їх у таблицю `assets_table` в PostgreSQL.
* **`quality`:** Запускається після успішного виконання `loader`. Перевіряє дані на пропуски, дублікати та генерує `quality_report.json`.
* **`research`:** Обчислює статистику, агрегує дані та генерує `research_report.json`.
* **`visualization`:** Створює набір графіків `.png` (гістограми, теплокарти, кругові діаграми) для відображення інсайтів.
